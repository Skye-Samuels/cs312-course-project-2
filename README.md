# Minecraft Server Deployment

This repository automates the provisioning and configuration of an AWS EC2 instance running a Minecraft server inside Docker.

---

## Background
This project replaces manual AWS console actions with an automated Infrastructure as Code (IaC) deployment pipeline. We use Ansible to create network resources and compute instances on AWS, patch the system host, install container dependencies, and manage the runtime environment.

---

## Pipeline Overview

1. **Local Initialization:** Reads AWS credential configurations from a local variables file.
2. **Infrastructure Provisioning:** Calls AWS APIs to create Key Pairs, Security Groups, and EC2 nodes.
3. **Configuration Management:** Patches the remote target system and installs the Docker Engine.
4. **Service Deployment:** Deploys Docker Compose templates and sets up a systemd wrapper for automated reboots and safe shutdowns.
5. **Infrastructure Teardown:** Automates the complete destruction of cloud resources to guarantee a clean slate.

---

## Requirements

* **Local OS:** Linux, macOS, or Windows Subsystem for Linux (WSL).
* **Tools:** Ansible Core, python-boto3, python-botocore.
* **Credentials:** Active AWS Learner Lab session keys.

---

## Deployment Instructions

### 1. Setup Local Environment Variables
Create a file named `.env` in the project root directory and paste your active AWS CLI credentials into it. Ensure this file is tracked by `.gitignore`.

### 2. Provision Infrastructure
Execute the provisioning script to create your cloud resources:
```bash
ansible-playbook provision.yml
```

### 3. Update Inventory Configuration
Open `inventory.ini` and replace the placeholder text with the public IP address generated by the previous command.

### 4. Deploy Application Stack
Execute the configuration script to install Docker and start the server:
```bash
ansible-playbook configure.yml
```

---

## Verification

Check the deployment and port availability using network scanning tools:
```bash
nmap -sV -Pn -p T:25565 <instance_ip>
```

Connect using your game client by targeting the instance IP address on port `25565`.

---

## Infrastructure Teardown

To completely destroy the provisioned AWS resources and ensure a clean environment, execute the teardown playbook:
```bash
ansible-playbook teardown.yml
```
